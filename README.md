# ZARAZ

Маркетплейс для бронювання послуг. Перший запуск: місто Стрий.

## Локальний запуск

Три термінали:

```bash
# 1. Локальна MongoDB (in-memory replica set, дані зберігаються в backend/.data)
cd backend && npm install && npm run db

# 2. Сідинг даних (один раз, поки термінал 1 запущений)
cd backend && npm run seed

# 3. Backend API
cd backend && npm run dev        # http://localhost:4000

# 4. Frontend
cd frontend && npm install && npm run dev   # http://localhost:3000
```

Відкрити http://localhost:3000 → редірект на /uk.

### Тестові акаунти (після `npm run seed`)

- Супер-адмін: `admin@zaraz.ua` / `SuperAdmin123!`
- Власники бізнесів: `<businessname>@example.com` / `BusinessOwner123!` (напр. `studionine@example.com`)

### Продакшн

Замінити в `backend/.env`:
- `MONGODB_URI` → MongoDB Atlas M0 (реплісет, підтримує транзакції)
- `CLOUDINARY_*` → реальний акаунт Cloudinary
- `SMTP_*` → реальний SMTP (напр. Gmail app password)
- `JWT_ACCESS_SECRET` / `JWT_REFRESH_SECRET` → нові випадкові 256-біт секрети

## Статус (Фаза 1)

- [x] Каталог з пошуком і фільтрами (домашня сторінка + /catalog)
- [x] Авторизація клієнта/бізнесу (JWT httpOnly cookies, email-підтвердження логується в консоль)
- [x] Dark/light mode, uk/en мультимовність
- [x] Профіль бізнесу (про нас, послуги, майстри, галерея-заглушка, рейтинг) — без реальних фото/карти/Google-відгуків
- [x] Система бронювання (вибір дати/слоту, MongoDB-транзакція, захист від подвійного запису, 409 при конфлікті)
- [x] Кабінет клієнта (записи з фільтром майбутні/минулі/скасовані, скасування з перевіркою політики, улюблені, сповіщення, профіль)
- [x] Кабінет бізнесу (дашборд зі статистикою, календар-тиждень з ручним записом і діями complete/no-show/cancel, послуги CRUD, майстри CRUD з редактором розкладу)
- [x] Супер-адмін (огляд платформи, підтвердження/відхилення нових бізнесів, блокування/розблокування)

### Свідомо не в цій фазі (за планом — Фаза 2/3)

- Галерея/карта/Google-відгуки на профілі бізнесу (потрібні реальні фото + Cloudinary + Leaflet)
- Відгуки клієнтів з модерацією (Фаза 2)
- Штрафи/ескалація за скасування — рейтинг клієнта поки не змінюється автоматично (Фаза 2)
- TOP-розміщення (замовлення й оплата), щомісячні рахунки комісії, cron-задачі (Фаза 2)
- Аналітика бізнесу, команда адмінів з правами, видимість телефону після часу запису (Фаза 3)
